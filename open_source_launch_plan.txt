--------------------------------------------------------------------------------
PART 1: GITHUB BRANCH PROTECTION WARNING
--------------------------------------------------------------------------------
The warning in your screenshot reads: "Your main branch isn't protected."

* Why this is shown:
  Currently, anyone with access can push code directly to the 'main' branch,
  rebase it, delete history, or override commits. As you transition DevHunt into
  a public open-source project, direct pushes risk breaking stable releases.

* How to resolve it:
  1. Click the "Protect this branch" button (or go to repository Settings -> 
     Branches).
  2. Click "Add branch protection rule".
  3. Under "Branch name pattern", type: main
  4. Enable these recommended settings:
     [x] Require a pull request before merging (Require approvals: 1)
     [x] Require conversation resolution before merging
     [x] Restrict who can push to matching branches (leave empty so only admin/PRs can merge)
  5. Scroll to the bottom and click "Create" or "Save Changes".
